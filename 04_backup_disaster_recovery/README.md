# Project 4 — Backup & Disaster Recovery: AWS & Azure

## Positioning

Self-Directed Cloud Infrastructure Project focused on backup planning, restore testing, business continuity and operational documentation.

## Objective

Demonstrate ability to design, document and test basic backup and disaster recovery procedures across AWS and Azure.

## Technologies

AWS:
- AWS Backup
- EBS snapshots
- S3 versioning
- S3 lifecycle rules

Azure:
- Recovery Services Vault
- VM backup
- Azure Storage redundancy
- Blob versioning where applicable

## What to build

Create backup and restore procedures for:
- Compute instance/VM
- Storage bucket/container
- Configuration documentation
- Recovery test

## RPO/RTO Example

RPO:
- Maximum acceptable data loss: 24 hours for lab resources.

RTO:
- Maximum target restoration time: 2 hours for lab resources.

These are lab assumptions and must be clearly labelled as such.

## Evidence to collect

Take screenshots of:
- AWS Backup plan
- EBS snapshot
- S3 versioning
- Azure Recovery Services Vault
- Azure backup item
- Restore test evidence
- RPO/RTO table
- DR runbook

Designed and documented a backup and disaster recovery lab across AWS and Azure. Configured backup concepts for compute and storage resources, documented RPO/RTO assumptions, created restore procedures and tested recovery steps in a controlled lab environment. The project demonstrates practical understanding of business continuity, cloud operations, infrastructure support and disaster recovery documentation.
