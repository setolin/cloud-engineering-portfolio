# Project 3 — Monitoring, Alerting & Incident Response: AWS & Azure

## Positioning

Self-Directed Cloud Operations Project focused on observability, alerting, incident triage and support runbooks.

## Objective

Build and document a monitoring and alerting workflow for AWS and Azure resources. The goal is to prove ability to detect issues, investigate alerts and follow structured support procedures.

## Technologies

AWS:
- Amazon CloudWatch
- CloudWatch Alarms
- CloudWatch Logs
- EC2 metrics

Azure:
- Azure Monitor
- Metrics
- Alerts
- Log Analytics Workspace

Optional:
- Grafana
- Prometheus

## Scenarios to simulate

1. High CPU usage
2. Disk or storage pressure
3. VM/instance stopped or unavailable
4. Failed access attempts or suspicious IAM activity
5. Latency or connectivity issue

## Evidence to collect

Take screenshots of:
- CloudWatch dashboard
- CloudWatch alarm
- CloudWatch logs
- Azure Monitor dashboard
- Azure alert rule
- Log Analytics query
- Grafana dashboard if used
- Incident runbook and test result

## LinkedIn Project Description

Built and documented a cloud monitoring and incident-response lab using AWS CloudWatch, Azure Monitor and operational runbooks. Configured alerts for compute and infrastructure conditions, simulated incidents such as high CPU usage and service unavailability, documented triage steps and produced support procedures to improve detection and response. This project demonstrates cloud operations, monitoring and structured troubleshooting skills relevant to Junior Cloud Engineer and Cloud Operations roles.
