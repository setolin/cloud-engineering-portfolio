# Project 1 — Secure Cloud Infrastructure Lab: AWS & Azure

## Positioning

Self-Directed Cloud Engineering Project focused on secure cloud infrastructure, networking, identity, monitoring readiness and operational support.

## Objective

Design and document a small but realistic cloud infrastructure environment using AWS and Azure. The purpose is to demonstrate practical understanding of infrastructure components that a Junior Cloud Engineer or Cloud Support Engineer must troubleshoot and support.

## Technologies

AWS:
- VPC
- Public and private subnets
- Route tables
- Security groups
- EC2
- S3
- IAM

Azure:
- Resource Groups
- Virtual Network
- Subnets
- Network Security Groups
- Virtual Machines
- Blob Storage
- Microsoft Entra ID / RBAC

Optional:
- Terraform
- PowerShell
- Bash

## Architecture

The project should include:
- One isolated AWS VPC with public and private subnet design.
- One Azure VNet with equivalent segmentation.
- One small compute instance/VM.
- Storage configured with restricted access.
- Security groups/NSGs configured with least-privilege inbound rules.
- IAM/RBAC roles documented.
- Basic operational runbook for troubleshooting connectivity.
