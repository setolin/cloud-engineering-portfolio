# Project 2 — IAM Security & Least-Privilege Access: AWS & Azure

## Positioning

Self-Directed Cloud Security and IAM Project designed to support a Cloud Engineer / Cloud Support Engineer profile with a security-focused foundation.

## Objective

Demonstrate practical understanding of identity, access management, least privilege, MFA, role-based access and basic audit controls across AWS and Azure.

## Technologies

AWS:
- IAM users
- IAM groups
- IAM roles
- IAM policies
- MFA
- Access Analyzer
- CloudTrail

Azure:
- Microsoft Entra ID
- Azure RBAC
- Role assignments
- Conditional Access concepts
- Activity logs

## What to build

Create a small access model with:
- Admin role
- Read-only support role
- Storage operator role
- Monitoring operator role
- Backup operator role

Restrict access based on job function. Avoid full administrator access except for the owner/admin account.

## Evidence to collect

Take screenshots of:
- AWS IAM policy
- AWS IAM role
- MFA status
- AWS Access Analyzer or CloudTrail event
- Azure RBAC assignment
- Azure Activity Log
- Documentation showing least-privilege reasoning

## LinkedIn Project Description

Designed and documented a least-privilege IAM model across AWS and Azure as part of a self-directed cloud security project. Created role-based access patterns for administration, monitoring, storage and backup operations, reviewed permissions, documented access risks and created an audit runbook for support and security checks. This project demonstrates practical IAM knowledge aligned with cloud support, infrastructure operations and security-focused engineering roles.
